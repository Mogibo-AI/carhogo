# ビジョン文書: CarHogo

## エグゼクティブサマリー

CarHogoは、ウェアラブルとAWSを連携させて、ドライバーに「プロアクティブ（先回り型）なAI介入」を提供する次世代ドライビングセーフティサービスです。

コンセプトは、その名の通り「過保護」です。眠気・怒り・遅刻ストレスという運転中の3大リスクを生体信号からリアルタイムで検知し、ドライバーが自覚する前にシステムが介入・解消します。

CarHogoが目指す究極の姿は、ドライバーから「自制心」や「忍耐」といった精神的負荷をすべて奪い去ることです。ただ前を向いてハンドルを握るだけの「運転マシーン（＝ダメ人間）」を生み出すほどの過保護さが、かつてない安全を実現します。

## ビジネスコンテキスト

### 問題提起

居眠りや感情的ストレス、スマートフォン操作に起因する運転事故は、依然として大きな社会課題であり続けています。

既存の対策の多くは、警告灯や車線逸脱アラートなどの「受動的」なもの、あるいはドライバー自身の意識に頼ったものです。それに対し、CarHogoは全く異なるアプローチを取ります。システムがドライバーに代わって介入することで、自己制御が最も困難になる瞬間に、そもそも自己制御を必要としない環境を提供します。

具体的な問題点は以下のとおりです。

- **居眠り運転**: ドライバーは眠気を感じても「まだ大丈夫」と過信しがちであり、自覚した時にはすでに反応速度が著しく低下しています。CarHogoは単なる警告に留まらず、強制的に意識を介入させることで、手遅れになる前の安全を確保します。
- **怒りと運転中のストレス**: 煽り運転や攻撃的運転は突発的な感情スパイクによって引き起こされます。感情が高まった瞬間に穏やかなAIの声が介入することで、運転行動に影響が出る前に感情の高まりを遮断します。
- **遅刻ストレスとスマートフォン操作**: 焦燥感に駆られたドライバーは、連絡のためにスマートフォンを操作し、重大な事故を招きます。CarHogoは外部への通知を自動化することで、ドライバーからスマートフォンを手に取る「理由」そのものを取り除きます。

### ビジネス推進要因

- **ウェアラブル×AWSの融合**: Pixel Watchから得られる高精度な生体データと、AWS IoT、生成AIをシームレスに統合。これにより、従来のシステムでは不可能だった「個人の状態に即応するリアルタイムな安全介入」を実現します。
- **Amazon Nova 2 Sonicの登場**: 超低遅延な音声生成AI「Amazon Nova 2 Sonic」の採用により、システムは単なる警告機を超え、ドライバーに寄り添う「対話型安全アシスタント」へと進化しました。自然な対話を通じて、心理面からも安全をサポートします。
- **自動車安全の新パラダイム**: 従来の安全技術が「注意喚起（気づき）」を終着点としていたのに対し、CarHogoは「行動の代行」を目的としています。このパラダイムシフトこそが、市場における最大の差別化要因となります。

### 対象ユーザーおよび関係者

| ユーザータイプ | 説明 | 主なニーズ |
|------------|------|----------|

| 同乗者・家族 | ドライバーの安全を気にかける人 | ドライバーが安全に目的地に到着するという安心感 |
| 企業・業務用車両管理者 | 社用車やタクシー・トラックなど業務用車両を管理する組織 | ドライバーの安全管理と事故リスクの低減 |

### ビジネス上の制約

- **デバイス依存**: 現時点ではPixel Watch（Wear OS）を前提とします。将来的には他のウェアラブルデバイスへの対応を検討します。
- **ネットワーク依存**: リアルタイム処理のためにモバイルデータ通信またはWi-Fi接続が必要です。
- **シングルユーザー（初期版）**: 初期バージョンは1名のドライバーを対象とします。将来的にはマルチユーザー対応を行います。

## フルスコープビジョン

### 製品ビジョンステートメント

CarHogoは、コネクテッドカーにおける「安全アシスタント」の標準を目指します。ドライバーの生体情報をリアルタイムに監視し、危険が顕在化する前に予兆を捉え、音声コミュニケーションを通じて自律的に介入します。CarHogoが実現する世界では、ドライバーの唯一の仕事はハンドルを握ることだけです。運転に必要なすべての「気遣い」「自制心」「忍耐」はCarHogoが代行します。ドライバーは究極の「ダメ人間」として、ただ安全に目的地へ到着することだけに集中できます。

### 機能エリア

#### 機能エリア1: 生体情報モニタリング

- **説明**: ウェアラブルデバイスからドライバーの生理的データを継続的に収集・分析します。
- **主な機能**:
  - 心拍数およびHRV（心拍変動）のリアルタイムモニタリング
  - 加速度計データによる動作・ジェスチャー検知
  - ドライバーごとのベースラインキャリブレーション
  - 眠気・ストレス・不安状態の異常検知
- **ユーザーにとってのメリット**: ドライバー自身が気づく前に、システムがドライバーの状態を把握します。

#### 機能エリア2: 居眠り防止（SLEEPアクション）

- **説明**: 居眠り運転状態の検知と強制的な覚醒介入。
- **主な機能**:
  - 一定時間継続する心拍数低下の検知
  - 低動作検知（ステアリング微調整の消失）
  - Amazon Nova 2 Sonicによる強制的なAI音声会話の開始
  - 心拍数がベースラインに戻るまで会話を継続
  - 状況に応じて動的に生成される会話トピック（ニュース・天気・雑談など）
- **ユーザーにとってのメリット**: ドライバーは単に警告を受けるのではなく、会話への参加によって覚醒状態を維持します。眠気に耐える努力すら不要にします。

#### 機能エリア3: 怒り・ストレス緩和（ANGERアクション）

- **説明**: 感情的な運転状態の検知と強制的な沈静化介入。
- **主な機能**:
  - 急激な心拍数スパイクの検知
  - 加速度計データによる身体的運動との区別
  - Amazon Nova 2 Sonicによる共感的なAI音声会話の開始
  - 全肯定的な会話スタイル（ドライバーの感情を受け止め、注意を転換）
  - 心拍数が安定するまで会話を継続
- **ユーザーにとってのメリット**: 「アンガーマネジメント」をシステムが代行します。ドライバーは怒りを自分でコントロールする必要がありません。

#### 機能エリア4: 遅刻連絡（LATEアクション）

- **説明**: 遅刻シナリオの自動検知とプロアクティブなコミュニケーション代行。
- **主な機能**:
  - Google Maps Directions APIを使用した現在地から目的地へのETA継続計算
  - Google Calendar APIによる出社時間（予定開始時刻）と目的地（場所フィールド）の取得。目的地はGoogleカレンダーに予め登録された場所情報を使用する
  - ETAが予定時刻を超過した時点での自動検知
  - Amazon Bedrockによるビジネス向け謝罪メッセージの自動生成
  - Amazon Nova 2 Sonicが生成したメッセージを音声で読み上げ、ドライバーに送信可否を確認する会話を行う
  - ドライバーが承認した場合、Amazon SNSによるSMS自動送信
  - ドライバーはスマートフォンに触れない
- **ユーザーにとってのメリット**: スマートフォン操作不要で「安全」と「謝罪」を両立します。遅刻の気まずさすらシステムが代行します。

#### 機能エリア5: CarHogoアプリケーション

- **説明**: ドライバーと同乗者のための主要な視覚インターフェースとなるブラウザベースのアプリケーション。運転中にダッシュボードに設置したタブレット・PCで常時表示します。
- **主な機能**:
  - リアルタイム生体情報表示（心拍数・HRV・ECG波形）
  - システムステータスとアクティブアクションの表示
  - Amazon Nova 2 Sonic音声会話UI（全アクション共通）
  - アクション履歴とログ
- **ユーザーにとってのメリット**: ドライバーと同乗者が現在のシステム状態を一目で把握できます。

#### 機能エリア6: IoTデータパイプライン

- **説明**: ウェアラブルからクラウドへのリアルタイムデータ取り込みとルーティング。
- **主な機能**:
  - Pixel WatchからAWS IoT CoreへのMQTT over TLS
  - イベントルーティングのためのIoT Rules Engine
  - クラウドとブラウザ間の状態同期のためのDevice Shadow
  - センサーからクラウドまでのサブ秒レイテンシ
- **ユーザーにとってのメリット**: システムは遅延なく生体変化にリアルタイムで反応します。

### 統合ポイント

- **Amazon Nova 2 Sonic** — リアルタイム双方向音声会話（SLEEP・ANGER・LATEアクション、全アクション共通）
- **Amazon Bedrock（Claude 3 Haiku）** — LATEアクションの遅刻メッセージテキスト生成
- **Google Maps Directions API** — LATEアクションのETA継続計算
- **Google Calendar API** — LATEアクションの出社時間（予定開始時刻）と目的地（場所フィールド）の取得
- **Amazon SNS** — LATEアクションのSMS自動送信（ドライバー承認後）
- **AWS IoT Core** — MQTTデータ取り込みとDevice Shadow管理

### ユーザー体験（フルビジョン）

#### ジャーニー1: 居眠りの検知と解消

1. ドライバーが高速道路を90分走行。心拍数がベースラインから18%低下し、ステアリングの微調整が消失。
2. CarHogoが居眠りパターンを検知。アプリがSLEEPアクションUIを起動。
3. Amazon Nova 2 Sonicが話しかける：「長距離お疲れ様です。今日の夕ご飯、何にするか決めてますか？」「突然ですが、クイズです。日本で一番長い川ってどこか知ってますか？」
4. ドライバーは声に驚いて応答。会話が自然に続く。
5. 4分間の会話後、ドライバーの心拍数がベースラインに回復。
6. CarHogoが回復を検知し会話を終了：「元気になってよかったです。安全運転で。」

**結果**: ドライバーは停車せずに安全に到着。眠気に耐える努力も、自分で気づく必要もなし。

#### ジャーニー2: 煽り運転の遮断

1. 高速道路で車が割り込んでくる。心拍数が8秒で72から98 BPMに急上昇。加速度計はドライバーが静止していることを示す。
2. CarHogoが怒りパターンを検知。アプリがANGERアクションUIを起動。
3. Amazon Nova 2 Sonicが話しかける：「今のはイライラしますよね。気持ちはよく分かります。あの車は病院に急いでいたのかもしれません。あなたの余裕が事故を防ぎました。」
4. ドライバーが息を吐いて応答。AIが共感的な会話を続ける。
5. 心拍数が76 BPMに回復。CarHogoが介入を終了。

**結果**: 感情の高まりが数秒以内に遮断。アンガーマネジメントをシステムが完全代行。

#### ジャーニー3: 遅刻連絡の自動処理

1. ドライバーが渋滞中。CarHogoがGoogleカレンダーから9時00分の会議と会議場所を取得。Google Maps APIでETAを9時22分と計算。
2. CarHogoが22分の遅延を検知。Amazon Bedrockが遅刻メッセージを生成。
3. Amazon Nova 2 Sonicがメッセージを音声で読み上げる：「おはようございます。現在交通渋滞のため、到着が9時22分頃になる見込みです。ご迷惑をおかけし大変申し訳ございません。このメッセージを送信してもよろしいですか？」
4. ドライバーが「はい、送って」と応答。
5. Amazon SNS経由でSMSが会議主催者に送信される。
6. アプリに送信済みメッセージのプレビューが表示される。
7. ドライバーはスマートフォンに触れない。

**結果**: 謝罪の気まずさすらシステムが代行。ドライバーは運転だけに集中。

### 拡張性と成長性

- **マルチドライバー対応**: 個別のベースラインと設定を持つ複数の登録ドライバーへの拡張。
- **生体情報の拡張**: より豊かな状態検知のためのEDA（皮膚電気活動）・皮膚温度・血中酸素濃度の追加。
- **車両統合**: 車両本体のシステムとの連携を実現して実際の車内エアコンやカーオーディオを直接制御。
- **業務用車両への対応**: 社用車やタクシー・トラックなど業務用車両を一括管理できるエンタープライズ版の提供。管理者が全ドライバーの安全状態をリアルタイムで把握。

## MVPの範囲

### MVPの目的

3つのCarHogoアクションがPixel Watchからの生体信号によって自律的にトリガーされ、Amazon Nova 2 Sonicによるリアルタイム音声会話とブラウザベースのアプリが連動して動作する、エンドツーエンドのサービスを実現します。

### MVPの成功基準

- [ ] 3つのアクション（SLEEP・ANGER・LATE）が生体信号または手動トリガー（開発者向けテスト機能）で正常に発動する
- [ ] SLEEPまたはANGERトリガーから5秒以内にAmazon Nova 2 Sonicの音声会話が開始する
- [ ] LATEトリガー時にNova 2 SonicがBedrockで生成したメッセージを読み上げ、ドライバーの承認後にSMSを送信する
- [ ] いずれかのトリガーから2秒以内にブラウザアプリが更新される
- [ ] システムが安定して継続動作する

### 対象機能（MVP）

| 機能 | 説明 | 優先度 | 含める理由 |
|------|------|--------|----------|
| Pixel Watch生体情報収集 | Health Services APIによる心拍数・HRV取得、MQTT送信 | 必須 | 全アクションのコアデータソース |
| SLEEPアクション: Nova 2 Sonic会話 | 眠気検知によってトリガーされるリアルタイム音声会話 | 必須 | 主要な差別化機能 |
| ANGERアクション: Nova 2 Sonic会話 | ストレス検知によってトリガーされるリアルタイム音声会話 | 必須 | 主要な差別化機能 |
| LATEアクション: Bedrock+Nova 2 Sonic連携 | Bedrockで生成した遅刻メッセージをNova 2 Sonicが読み上げ、ドライバーの承認後にSMS送信 | 必須 | AIテキスト生成と音声会話の統合実証 |
| ブラウザベースのCarHogoアプリ | リアルタイム生体情報表示・アクションステータス・Nova 2 Sonic UI | 必須 | 主要な視覚インターフェース |
| AWS IoT Coreパイプライン | MQTTデータ取り込み・IoT Rules・Device Shadow | 必須 | コアインフラ |
| 生体情報解析ロジック | 眠気・怒り検知アルゴリズム | 必須 | アクションのトリガー |
| AWS CDKインフラ | コードとして定義された全AWSリソース | 必須 | 再現可能なデプロイ |

### 明示的に対象外となる機能（MVP）

| 機能 | 延期理由 | 対象フェーズ |
|------|---------|------------|
| 実際の車内エアコンやカーオーディオ制御 | 車両本体のシステムとの連携が必要 | フェーズ3 |
| マルチドライバー対応 | 初期版はシングルユーザー | フェーズ2 |
| 加速度センサーの完全実装 | 心拍数が主要シグナル。加速度は補助的 | フェーズ2 |
| GPS速度連携 | 渋滞判定の精度向上に有効だが必須ではない | フェーズ2 |
| 業務用車両への対応 | エンタープライズ機能 | フェーズ3 |

### MVPの制約と前提

- **前提**: Amazon Bedrock Claude 3 Haikuのアクセスが承認済み。
- **許容される制限**: 加速度センサーの値は初期版では固定値を使用。心拍数が主要な検知シグナル。

### MVPの完了の定義

- [ ] 必須機能がすべて実装・テスト済み
- [ ] Nova 2 Sonic会話がトリガーから5秒以内に開始する
- [ ] ブラウザアプリがトリガーから2秒以内に更新される
- [ ] エンドツーエンドの動作確認が完了している

## リスクと依存関係

### 主なリスク

| リスク | 発生可能性 | 影響度 | 軽減策 |
|--------|----------|--------|--------|
| Amazon Bedrockアクセスが承認されない | 中 | 中 | フォールバックテキストライブラリを実装。即座にアクセス申請 |
| Pixel Watch MQTT接続が不安定 | 中 | 高 | 接続安定性を事前テスト。手動トリガー機能をフォールバックとして実装 |
| Nova 2 Sonicのレイテンシが5秒を超える | 低 | 中 | 早期にレイテンシをテスト。プロンプトと接続設定を最適化 |

### 外部依存関係

- Amazon Nova 2 Sonic — AWS
- Amazon Bedrock Claude 3 Haiku — AWS（モデルアクセスの申請と承認が必要）
- Google Maps Directions API — Google
- Google Calendar API — Google
- Amazon SNS — AWS（SMS送信）
